# tgianella.dev

Source code for my personal website at [tgianella.dev](https://tgianella.dev).
A bilingual (EN/FR) site with a blog, conference talks, and a collection of pictures.

## Tech stack

- [Astro 6](https://astro.build) -- static site generator with MDX support
- [Open Props](https://open-props.style) -- CSS design tokens
- Native CSS (no preprocessor), BEM naming, mobile-first
- Full i18n (English + French) with type-safe translation dictionaries
- View Transition API for SPA-like navigation with directional animations
- Images served from a Cloudflare R2 CDN

## Getting started

```bash
pnpm install
pnpm dev          # dev server at localhost:4321
```

## Commands

| Command            | Description                             |
| :----------------- | :-------------------------------------- |
| `pnpm dev`         | Start the dev server                    |
| `pnpm build`       | Production build to `./dist/`           |
| `pnpm preview`     | Preview the production build            |
| `pnpm astro check` | TypeScript + Astro type checking        |
| `pnpm test`        | Run unit tests (Node's built-in runner) |
| `pnpm lint`        | ESLint (TS + Astro + a11y)              |
| `pnpm lint:style`  | Stylelint (CSS + Astro styles)          |
| `pnpm format`      | Prettier                                |

## Project structure

```
src/
  components/      Reusable Astro components
  content/         MDX content collections (blog, talks, gallery)
    blog/{en,fr}/    Blog articles
    talks/{en,fr}/   Conference talks
    gallery/{en,fr}/ Photo entries (images hosted on CDN)
  i18n/            Translation dictionaries and locale utilities
  layouts/         BaseLayout (shared shell) and MainContent wrapper
  pages/           File-based routing ([lang]/blog/[slug], etc.)
  scripts/         Client-side scripts (view transitions, scroll, etc.)
  styles/          Global CSS and design tokens
public/
  fonts/           Self-hosted Bitstream Charter
  theme.js         Blocking theme script (FOUC prevention)
  _headers         Security headers (CSP, HSTS, etc.)
docs/
  ARCHITECTURE.md  Technical decisions and non-obvious patterns
```

## Content

Content is organized in three collections. Both English and French versions of
each piece share the same slug (e.g., `en/my-article.mdx` and
`fr/my-article.mdx`).

**Blog** -- MDX articles with metadata (date, tags, reading time, optional
co-authors).

**Talks** -- Conference presentations with event history, embedded YouTube
replays, and slides. The talks page includes a timeline view that groups talks
by event.

**Gallery ("Blue Screens")** -- Photos of public display screens showing errors,
desktops, or other unintended content. Images are hosted externally; thumbnails
are generated by URL convention.
