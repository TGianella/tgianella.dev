title: "How do you rate yourself in responsive CSS ?"
description: "An overview of the many available tools to make a web page responsive"
abstract: |
    Responsive CSS is something we all know, right ? Since smartphones are a thing, user interfaces have to adapt to every possible screen size, from 4K screens to tiny smartphones.
    How do you achieve a responsive page ? In the beginning, we only had our trusty breakpoints and an army of media queries. Then new tools arrived: flexbox, grid, relative units... Today with container queries and ever more units (dvmin, cqh...), we have a variety of tools and techniques at hand and few are those able to claim that they master them all. So, how do you rate yourself in responsive CSS ?
    Whether it's 1 or 10, come (re)discover with us all the ways to create responsive webpage nowadays: we cooked some mock-up designs, from the simplest to the most complex, and we are going to implement them live to demonstrate all these techniques.
    We'll see together that for once, things were not better before!
